# 유형 3 — 도입 타당성형 (Feasibility & Risk)

**용도**: 신기술·신제도·신규 프로세스 도입 검토, 리스크 평가, 규제 검토, Go/No-Go 판단
**전형적 상황**: "우리 회사에 이걸 도입해도 되나", "어떤 리스크가 있고 어떻게 막나", "선행 조건은"

## 컨텍스트 수집 (부족한 것만 질문)

1. 도입 대상과 적용 범위는? (전사 / 특정 부서 / 파일럿)
2. 도입으로 해결하려는 문제는? (문제 정의가 약하면 그것부터 리서치 범위에 포함)
3. 조직 제약은? (예산, 인력 역량, 기존 시스템, 노조/문화)
4. 규제·보안 민감도는? (개인정보, 산업 규제, 사내 보안 정책)
5. 실패 시 최악의 시나리오는? (되돌림 가능성)

## 기본 MECE 프레임

- **차원**: 도입 옵션/시나리오 (전면 도입 / 단계 도입 / 파일럿 / 미도입+대안)
- **관점**: ① 기대 효과(정량 근거) ② 기술 리스크 ③ 보안·법규 리스크 ④ 조직·변화관리 리스크
  ⑤ 선행 조건(인프라, 역량, 거버넌스) ⑥ 비용(도입+운영+숨은 비용) ⑦ 유사 조직 도입 사례

## 브리프 작성 시 요구할 분석 방식

- 기대 효과 주장의 근거를 "벤더 주장 / 제3자 검증 / 학술 연구"로 등급 구분
- 실패 사례를 성공 사례와 같은 비중으로 수집하고 실패의 공통 원인 추출
- 각 리스크에 발생 가능성 × 영향도 평가와 구체적 완화 방안 연결
- 규제는 현행 기준과 입법 예고/개정 동향을 구분해 조사
- 도입하지 않을 경우의 비용(기회비용, 경쟁 열위)도 대칭적으로 분석

## 출력 규격

1. Go/No-Go 판단 프레임 (판단 기준 + 각 기준의 현재 충족도)
2. 리스크 매트릭스 (가능성×영향도, 각 리스크별 완화 방안과 담당 영역)
3. 단계별 도입 로드맵 초안 (선행 조건 → 파일럿 → 확산, 각 단계 성공 기준)
4. 유사 조직 사례 3~5개 (규모·산업 유사성 명시, 실패 사례 최소 1개 포함)

## 검증 루프 (2차 프롬프트)

- "이 도입을 반대하는 CISO/법무/노조 관점에서 가장 강한 반론을 구성하라"
- "완화 방안 각각이 실제로 작동하지 않은 사례가 있는지 확인하라"
- "선행 조건 중 우리가 과소평가하기 쉬운 것은 무엇인가 (특히 조직 역량)"
