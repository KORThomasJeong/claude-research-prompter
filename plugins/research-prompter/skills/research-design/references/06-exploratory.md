# 유형 6 — 탐색·학습형 (Exploratory Learning)

**용도**: 새 개념·분야를 체계적으로 이해, 지식 기반 다지기, Obsidian 노트용 개념 정리
**전형적 상황**: "OO를 제대로 이해하고 싶다", "이 분야 입문", "빠르게 전문가 수준 감 잡기"

## 컨텍스트 수집 (부족한 것만 질문)

1. 대상 개념/분야와 현재 이해 수준은? (전혀 모름 / 개요는 앎 / 실무 적용 목적)
2. 학습의 최종 목적은? (교양 / 업무 적용 / 다른 지식과 연결 / 남에게 설명)
3. 투입 가능한 시간은? (한 세션 / 몇 주 커리큘럼)
4. 선호하는 이해 방식은? (역사적 발전 순 / 문제 중심 / 제1원리부터)

## 기본 MECE 프레임

- **차원**: 개념의 구성 요소 / 하위 분야 (리서치가 먼저 분해 구조를 제안)
- **관점**: ① 정의와 역사적 맥락(왜 등장했나) ② 핵심 원리·메커니즘 ③ 주요 논쟁점·학파 차이
  ④ 핵심 문헌·인물 ⑤ 인접 개념과의 관계(구분과 연결) ⑥ 실무 적용·최신 동향

## 브리프 작성 시 요구할 분석 방식

- 개념을 낳은 문제부터 설명 (해답보다 질문을 먼저)
- 흔한 오해(misconception) 목록과 왜 그런 오해가 생기는지 설명
- 전문가들 사이의 이견 지점을 명시 (합의된 것 vs 논쟁 중인 것 구분)
- 비유·사례로 설명하되 비유의 한계도 명시
- "이해했는지 확인하는 질문" 5개를 함께 생성 (자가 테스트용)

## 출력 규격

1. 개념 지도 (핵심 개념과 관계, Obsidian 링크 구조로 변환 가능한 형태)
2. 단계별 학습 로드맵 (기초 → 심화 → 논쟁, 각 단계 예상 시간)
3. 핵심 자료 10선 (원전/입문서/최신 리뷰 구분, 접근 가능한 것 위주)
4. 자가 점검 질문 5개 + 다른 지식과 연결할 접점 제안

## 검증 루프 (2차 프롬프트)

- "위 설명에서 단순화 때문에 왜곡된 부분이 있다면 정확한 버전을 제시하라"
- "이 분야 전문가가 위 요약을 보면 지적할 부분은?"
- "이 개념이 실제로 실패하거나 적용되지 않는 경계 조건은?"
