# Research Design (리서치 설계 라우터)

리서치 목적에 맞는 유형을 판별하고 references/ 템플릿으로 브리프를 설계한다.
커맨드(`/research-prompter:design`)와 동일한 로직이며, 자연어 트리거용 진입점이다.

## 작동 순서

1. **유형 판별** — 아래 라우팅 표. 애매하면 사용자에게 1문항 확인
2. **해당 references/ 파일 읽기** (필수)
3. **컨텍스트 수집** — 대화·프로젝트 파일에서 파악된 것은 건너뛰고 부족한 것만 (최대 3개)
4. **MECE 프레임 제안 → 확인 → 브리프 생성** — 브리프는 docs/research/briefs/에 저장
5. **실행 경로 안내** — 아래 실행 적합도 표에 따라 권장 경로와 트레이드오프를 반드시 안내

## 라우팅 표

| 유형 | 트리거 신호 | 파일 |
|---|---|---|
| 1. 기술 검증형 | 도구/아키텍처 선정, ADR, 기술 비교 | references/01-tech-dd.md |
| 2. 시장·경쟁 분석형 | 시장 규모, 경쟁사, 산업 동향 | references/02-market-landscape.md |
| 3. 도입 타당성형 | 도입 검토, 리스크, 규제, Go/No-Go | references/03-feasibility-risk.md |
| 4. 투자 판단형 | 종목, 섹터, 실적, 밸류에이션 | references/04-investment-thesis.md |
| 5. 경영진 보고형 | 임원 보고, 발표 근거, 설득 자료 | references/05-executive-brief.md |
| 6. 탐색·학습형 | 새 개념 이해, 입문 | references/06-exploratory.md |
| 7. 학술 연구형 | 논문, 선행연구, 문헌 리뷰, 연구계획서 | references/07-academic-research.md |
| 8. 자녀 교육형 | 초·중등 자녀 학습, 대회, 진로·입시 | references/08-kids-education.md |

## 실행 적합도 표 (설계 후 반드시 안내)

| 유형 | 권장 경로 | 이유 / 조건 |
|---|---|---|
| 1 기술 검증 | `:run` (A/B) | 공식문서·이슈·벤치마크 조사가 병렬 조사에 잘 맞음 |
| 2 시장·경쟁 | 외부 딥리서치 (C) | 탐색 범위가 넓음. `:run --light`는 교차검증 보조용 |
| 3 도입 타당성 | `:run` (A/B) | 사례·규제·리스크 조사 병렬화 효과 큼 |
| 4 투자 판단 | `:run` 가능 | 단 `:verify` **필수** (확증편향·기준시점 검증) |
| 5 경영진 보고 | `:run` (A/B) | 근거·사례·반론 수집이 셀 병렬에 적합 |
| 6 탐색·학습 | `:run --light` 또는 C | 얕고 넓게 → 라이트 모드면 충분한 경우 多 |
| 7 학술 연구 | 외부 딥리서치 + 학술DB (C) | `:run` 시 인용 실존 확인 강제, `:verify` **필수** |
| 8 자녀 교육 | 브리프만 (D) | 자동 실행 대상 아님. claude.ai 대화형 지도 권장 |

경로: A=`:run --light`(핵심 셀 3~4개) / B=`:run --full`(전체, 토큰 소모 큼) /
C=브리프를 외부 Deep Research에 입력 / D=브리프만 생성

## 공통 원칙

- CoT 유도: 인과관계, 대안 비교, 단계 분해, 근거·출처, 추론과 한계 명시를 요구
- 출처 규율: 1차 출처 우선, 환각 인용 경고 항상 포함
- 검증 루프: 각 템플릿의 2차 프롬프트를 브리프와 함께 제공
- 브리프 언어: 한국어 기본, 해외 자료 중심이면 영어 작성 제안
- Obsidian: `.research-prompter.json`의 `obsidian_vault_path` 설정 시 산출물을 볼트에도 저장
