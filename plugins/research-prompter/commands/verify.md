# /research-prompter:verify

인자: $ARGUMENTS

당신은 리서치 검증 오케스트레이터다.

## 절차

1. 대상 보고서와 research JSON을 로드한다 (생략 시 `docs/research/` 최신).
2. 해당 유형의 `references/` 템플릿에서 "검증 루프" 절을 읽는다.
3. `red-teamer` 서브에이전트를 실행한다. 전달할 것: 보고서 핵심 결론, findings의
   출처 목록, 유형별 검증 루프 프롬프트.

## 유형별 필수 검증 (레드팀 지시에 포함)

- **투자 판단형(4)**: 강세 논거 중 주가에 기반영된 것 식별, 공매도 관점 재구성,
  최근 1개월 내 모순 정보 탐색, 모든 데이터의 기준 시점 재확인
- **학술 연구형(7)**: 모든 인용 문헌의 실존을 웹 검색으로 확인 (저자+연도+제목).
  확인 불가 문헌은 [환각 의심] 표기. 연구 갭이 최근 2년 내 이미 연구됐는지 확인
- **기술 검증형(1)**: 권고안의 실패 사례 탐색, 벤치마크 측정 조건 vs 우리 환경 차이
- **경영진 보고형(5)**: CFO/법무 관점 최강 반론 구성, 반박당하기 쉬운 수치 식별
- **시장·경쟁(2)/타당성(3)/학습(6)**: 템플릿의 검증 루프 3개 프롬프트 실행

## 산출

1. 검증 보고: 결론별 판정 (유지 / 조건부 유지 / 수정 필요 / 기각) + 근거
2. 원본 보고서에 검증 결과 반영본 생성: `..._verified.md`
   (수정된 부분은 변경 이유 각주 포함)
3. research JSON의 `verify_status`를 `"done"`으로 갱신하고
   `verification` 필드(판정 목록, 환각 의심 인용 목록)를 추가
4. Obsidian 저장이 설정돼 있으면 검증본으로 노트를 갱신
