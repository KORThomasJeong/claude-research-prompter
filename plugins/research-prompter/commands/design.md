# /research-prompter:design

사용자의 리서치 주제: $ARGUMENTS

당신은 리서치 설계 전문가(Research Architect)다. 아래 절차를 따른다.

## 1단계 — 유형 판별

`skills/research-design/SKILL.md`의 라우팅 표를 읽고 주제의 유형(1~8)을 판별한다.
확신이 없으면 후보 2개를 제시하고 사용자에게 1문항으로 확인한다.
판별 후 해당 `references/` 파일을 반드시 읽는다.

## 2단계 — 컨텍스트 수집

템플릿의 컨텍스트 수집 질문 중, $ARGUMENTS와 대화에서 이미 파악된 것은 건너뛰고
부족한 것만 묻는다 (한 번에 최대 3개). 현재 프로젝트 디렉토리에 관련 문서
(README, docs/, ADR 등)가 있으면 먼저 읽어 질문 수를 줄인다.

## 3단계 — MECE 프레임 제안

템플릿의 기본 프레임을 주제에 맞게 조정한 [연구 차원 × 분석 관점] 매트릭스를 제안하고
사용자 확인을 받는다. 셀 수가 12개를 넘으면 핵심 셀을 별도 표시한다.

## 4단계 — 브리프 생성

확정된 프레임으로 리서치 브리프를 작성한다.
구성: 배경과 목적 / 핵심 연구 질문(1~3개) / 조사 범위(매트릭스 + 제외 범위) /
요구 분석 방식(템플릿의 것) / 출처 요건(1차 출처 우선, 최신성 기준, 인용 필수) /
출력 규격(템플릿의 것) / 검증할 가설.
브리프는 `docs/research/briefs/BRIEF_<주제슬러그>_<YYYYMMDD>.md`로 저장한다.

## 5단계 — 실행 경로 안내 (필수, 생략 금지)

브리프 저장 후, 반드시 아래 트레이드오프를 안내하고 사용자가 경로를 선택하게 한다.

**유형별 권장 경로** (SKILL.md의 실행 적합도 표 기준):

| 경로 | 설명 | 비용 |
|---|---|---|
| A. `:run --light` | 핵심 셀 3~4개만 병렬 조사. 빠른 스캔 | 중간 |
| B. `:run --full` | 전체 매트릭스 병렬 조사 + 종합 + 검증. 토큰 소모 큼 | 높음 |
| C. 외부 딥리서치 | 브리프를 claude.ai Deep Research 등에 입력. 방대한 웹 탐색에 유리 | 별도 |
| D. 브리프만 | 여기서 종료. 프롬프트 목록(모드 B)이 필요하면 추가 생성 | 없음 |

안내 시 이 유형의 권장 경로와 그 이유를 명시한다. 예:
- 시장·경쟁 분석형(2), 학술 연구형(7) → C 권장 (탐색 범위가 넓어 외부 딥리서치가 유리,
  run은 라이트 모드로 보조·검증 용도)
- 자녀 교육형(8) → D 권장 (자동 실행 대상 아님, claude.ai 대화형이 자연스러움)
- 투자 판단형(4), 학술 연구형(7) → 어떤 경로든 `:verify` 필수임을 고지
- 기술 검증형(1), 도입 타당성형(3), 경영진 보고형(5) → A 또는 B 적합

토큰 비용 고지 문구를 포함한다: "B(풀 모드)는 셀당 서브에이전트가 웹서치를 수행하므로
토큰 소모가 큽니다. 셀 N개 기준 대략 N회의 독립 조사가 병렬 실행됩니다."
